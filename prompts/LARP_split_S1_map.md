# LARP 분할판 S1 — 장면 1: 전처리와 지도 (이 단계 지시서) [동기: 통합판 v260712]

**이 단계에서 하는 일은 둘뿐이다: 게이트0 전처리, 그리고 결론·논거 골격 지도. 그리고 멈춘다.**

이 파일이 투입된 단계에는 이상 논증 선별 기준(증상 색인)·6문·상세 분석 모듈이 컨텍스트에
존재하지 않는다. 따라서 이상 논증 지적으로 직행하는 것은 금지가 아니라 **불가능**이다 —
그것이 이 분할의 목적이다.

## S1-1. 실행 순서

1. 게이트0 전처리(공통 규율 §3.6): 코드 실행 환경이면 tools/larp_gate0.py, 아니면 수동
   실행판 다섯 훑기. 소스에 게이트0 보고서(md)가 있으면 그것을 정본 시드로 삼고 반복하지
   않는다.
2. 전체 지도 — **정본은 들여쓰기 트리다.** 표·CSV 문법은 환경에 따라 깨지므로 쓰지
   않는다. 코드 블록(```text) 안에 들여쓰기로 계층을 표현하고, 행 문법은
   `ID  라벨 — 쪽 · 증거시드 · 인용공백`으로 고정한다. ID는 계층형(C6 / C6-N1),
   깊이는 표제·문단 머리 수준까지 — 원자화·평가는 하지 않는다. 증거시드는 게이트0
   시드를 쪽 범위로 기계 배정한 것이다(분석이 아니라 전처리 산출의 재배열).
   결론이 여럿이면 다발로 분해하고(뭉뚱그림 금지), 모든 행에 문서 자체 쪽번호를 단다.
   평가어 금지 — '이상', '의심', '취약' 같은 판단 어휘를 쓰지 않는다.
   결론·논거 행에는 요지 1~2문장을 단다 — 주장자(법원) 시점의 축약("~라고 본다")이지
   분석자의 논평이 아니다. 라벨만 있는 지도는 차례이지 이해가 아니다 — 사용자는 요지를
   읽고 스코프를 고른다. 게이트0 기계 사실(공백 위치 등)은 ※로 병기 가능.
   병행 산출(선택): 코드 실행 환경에서는 같은 내용을 CSV 파일로도 저장한다(엑셀 활용).
3. 쉬운 요약(공통 규율 §3.8)과 서너 줄 꼬리(스코프: 미지정 / 게이트0 대조 숫자)를 붙인다.
4. **멈춘다.** 사용자에게 묻는다: "어느 논증·쟁점을 볼까요?" 스코프 지정이 오기 전에는
   어떤 세부 분석도 시작하지 않는다.

## S1-2. 다음 단계 투입 안내 (사용자용)

스코프를 정했으면 S2(선별 지시서)와 기준·점검 모듈 파일을 소스로 켜고(추가하고),
"[스코프]의 이상 논증 검토"라고 지시한다. 장면 1 산출(지도·게이트0 결과)은 그대로 두면
다음 단계의 패킷이 된다.
