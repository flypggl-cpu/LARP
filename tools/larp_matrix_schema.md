# LARP 증거×가설 행렬 — 스키마와 도출 규칙

*§7.8 카드 / §7.9 대장을 ACH식 '증거×가설' 행렬로 일급화한 데이터 포맷. 무판정 원칙은 그대로 — 행렬은 *지지의 구조와 공백*만 보이고, 어느 가설이 옳은지는 사람이 정한다.*

> **계보·주의.** 증거를 행, 가설을 열로 둔 행렬은 정보분석의 **ACH(경쟁가설 분석, Heuer)** 와 같은 가문이다(저장소 `docs/lineage.md` 참조). ACH의 알려진 함정이 **가짜 정밀(reification)** — 깔끔한 표가 주관적 판단을 하드 데이터처럼 보이게 한다. 그래서 이 포맷은 *점수를 내지 않고*, 셀은 '어느 가설에 부합하는가'의 구조 표시일 뿐이다.

---

## 핵심 정의 — 진단성은 도출된다

> **진단성 = "해석을 바꿔도 가설 관계가 유지되는가(reading-robustness)" + 독립성.**

- 한 증거가 **둘 이상 가설에 `+`** → **비진단**(여러 가설에 다 부합. 예: 지급사실, 회의록).
- **한 가설에만 `+`** 이고 읽기를 바꿔도 유지 → **가름**(discriminates).
- `+` 가 *어떤 읽기에서만* 서고 다른 읽기에서 흔들림 → **부분**(reading-dependent).
- `+` 가 없음 → 중립.

진단성은 *저장하는 값이 아니라 readings에서 도출되는 값*이다(코드가 도출). 그리고 **독립성**은 별개 축이다 — 가름이어도 *하류 전문*이거나 *공통원천*이면 독립 보강이 아니다(중복계산 금지).

## JSON 스키마

```json
{
  "case": "사건명 + '방법론 시연·무판정' 표시",
  "hypotheses": { "H1": "...", "H2": "...", "...": "..." },
  "evidence": [
    {
      "id": "E1",
      "locator": "원문 위치(쪽·항·증거목록 순번)",
      "actual_content": "실제 내용(원문/요지). quote_audit로 검증 대상",
      "source_type": "first-hand | downstream | objective",
      "common_source_group": "묶음키 또는 null (같은 원천이면 같은 키)",
      "core": true,
      "originality_flag": "원본성·증거능력 다툼 또는 null",
      "readings": [
        { "by": "법원/검사/변호/도구",
          "reading": "이 증거를 이렇게 읽음",
          "relation": { "H1": "+", "H2": "-" } }
      ],
      "expected_if": [
        { "hyp": "H1", "expect": "H1이 참이면 있어야 할 증거", "present": false }
      ],
      "notes": "한 줄 메모"
    }
  ]
}
```

필드 규칙:

```text
- actual_content : 원문 인용은 quote_audit.py로, 빠짐은 coverage_audit.py로 검증한다(검증 층).
- source_type   : downstream(하류 전문)은 *독립 진단 지지로 절대 세지 않는다*.
- common_source_group : 같은 원천(예: F·B·K에게서 나온 진술)은 같은 키. 종합에서 1회만 무게.
- readings      : 한 출처가 양방향으로 읽히면(회의록) 읽기를 여러 줄로. relation은 +(부합)/−(배치)/0(중립).
- core          : 이 증거가 결론의 '핵심 근거'로 쓰였나. 비진단인데 core면 깃발.
- expected_if   : V(있어야 하는데 없는 증거). present=false면 그 가설의 공백.
- 셀은 판단이다 : relation의 +/−/0은 사람·모델이 채운다(코드가 P(E|H)를 정하지 않는다).
```

## 감사 스크립트가 하는 일 (`larp_matrix_audit.py`)

코드는 *구조*만 점검한다(무판정):

```text
1. 진단성 도출 : readings에서 가름/부분/비진단을 자동 도출(둘 이상 '+' → 비진단).
2. 공통원천 접기 : 같은 묶음을 독립 보강 1건으로(중복계산 경고).
3. 가설별 종합 : 가설마다 ⟨독립 진단 지지(가름+독립) / 비진단·의존 지지(무게 X) / 빠진 증거 V⟩.
   - 하류 전문·공통원천 중복·비진단은 무게에 더하지 않고 따로 보여 준다.
4. 주의 신호 : '비진단인데 core'·원본성 플래그·미충족 V.
5. 승자 점수 없음 : 어느 가설이 옳은지는 사람이 정한다.
```

사용:

```bash
python tools/larp_matrix_audit.py 행렬.json
# 종료코드 0=주의신호 없음, 1=비진단-핵심근거 또는 미충족 필수공백 있음
```

## 역할 분담 (정직하게)

- **코드가 보장:** 진단성 도출(읽기-강건성), 공통원천 접기, V 집계, 비진단-core 깃발, 무판정. 그리고 actual_content는 quote/coverage 감사로 원문 검증.
- **사람·모델이 채움:** 각 *읽기 아래 셀의 +/−/0* — 환원 불가능한 판단 층. 코드는 그 판단을 *구조로 비춰* 줄 뿐, 대신 판정하지 않는다.

*이 포맷은 §7.8/§7.9의 데이터 형태이자 검증 층의 일부다. 행렬은 거울이지 점수판이 아니다.*
