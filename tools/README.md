# tools — 검증 층 코드 (누락·환각·완전성 검사)

*한국어 | [English](README.en.md)*

이 폴더는 전체판 LARP의 **검증 층(LARP.md §3.7)의 코드 부분**입니다. AI가 낸 1차 분석을 사람이 다시 읽으며 검토해도 **눈으로는 걸러내기 어려운 위험**(지어낸 인용은 진짜와 똑같아 보이고, 빠뜨린 증거는 '없는 것'이라 눈에 안 띄죠)을 **매번 똑같이·기계적으로 짚어내 *눈에 보이게*** 만드는(즉 결정론적으로) 보조 도구가 들어 있습니다. 각 도구가 정확히 *무엇을 하는지*를 아래 표로 정리하고, 실행법(모두 공통)과 상세 사용서는 그 아래에 있습니다.

> **왜 AI 스스로는 못 잡나?** AI는 문서를 *찾아보는* 사서가 아니라, 그럴듯한 답을 *만들어내는* 이야기꾼에 가깝습니다. 그래서 긴 글의 증거를 빠뜨려도(완전성), 원문에 없는 인용을 지어내도(충실성) 스스로는 알아채지 못하고, "빠뜨린 것 없냐"고 같은 AI에게 되물어도 같은 자리를 또 놓칩니다 — 두 번째 눈이 아니라 *같은 눈*이니까요. **"AI가 만들었다"는 "찾아서 확인했다"가 아니라서**, 아래 도구들이 그 확인을 기계로 대신합니다. (자세히: [AI의 한계](../docs/ai_problems_and_this_tool.md))

> **모드 중립.** 이 도구들은 특정 모드 전용이 아닙니다 — 기본 LARP-Map·긴 문서 모드·전체판 LARP, *어느 모드가 그린 지도/출력에도* 그대로 대조해 쓸 수 있습니다.

## 검증 검사 스크립트 — 잔여 위험을 막는다

전체판 1차 출력에는 사람이 보고도 못 거르는 두 위험이 남습니다 — **침묵의 누락**(안 올라온 증거·약점)과 **위장된 환각**(원문 인용처럼 보이는 지어낸 문장). 이 폴더의 스크립트들이 그 검증 층의 코드 부분이며, 모델의 성실성에 기대지 않고 위험을 매번 똑같이·기계적으로 짚어내 *눈에 보이게* 만듭니다(즉 결정론적으로).

| 스크립트 | 잡아내는 문제 | 쉽게 말하면 |
|---|---|---|
| `larp_coverage_audit.py` | 빠뜨린 증거 | 글이 번호(꼬리표)로 인용한 증거를, 내 지도가 하나라도 빠뜨리지 않았는지 기계로 셉니다 |
| `larp_card_audit.py` | 대충 뭉뚱그린 평가 | 증거를 하나씩 따로 따졌는지 — 여러 개를 뭉뚱그리거나 평가 칸을 비워 두지 않았는지 봅니다 (§7.8 카드·§7.9 대장) |
| `larp_quote_audit.py` | 지어낸 인용 | "원문에 이렇게 써 있다"며 든 문장이 정말 원문에 있는지 글자 그대로 대조합니다 |
| `larp_matrix_audit.py` | 쓸모없는 증거·중복 계산 | 어느 편도 못 가르는 증거를 핵심 근거로 삼았는지, 같은 출처를 여러 번 세었는지, 빠뜨린 가설은 없는지 봅니다 (증거×가설 행렬 §7.10) |
| `larp_recon0_audit.py` | 장부 계산 오류 | "이 확신이 어디서 왔나"를 적은 장부의 숫자 합·표기가 앞뒤로 맞는지 검산합니다 (대조0 §7.10, 형식 `larp_recon0_schema.md`) |
| `larp_stat_audit.py` | 안 맞는 통계 수치 | 논문이 내놓은 통계값(p·t·χ²·신뢰구간)을 다시 계산해 앞뒤가 맞는지, 아예 나올 수 없는 값은 아닌지 대조합니다 (논문·통계용, 무판정) |

> **개선 루프용(개발자용):** `larp_recall_audit.py` 는 위 검사들과 쓰임이 다릅니다 — 미리 만든 정답 모음(골드셋 `gold_set.json`)과 비교해, 새 출력이 놓친 것 없이 얼마나 잡아냈는지(*재현율*)를 점수로 매깁니다. 도구를 *고쳐 갈 때* 성능이 떨어지지 않았는지 재는 용도지, 개별 분석에 쓰는 검사가 아닙니다.

의미적 누락(꼬리표 없는 약점·반론)은 코드로 다 못 잡으므로, 별도 모델 패스 [`LARP_verify.md`](../prompts/LARP_verify.md)(누락 사냥)와 함께 씁니다. 전체 순서는 [USAGE의 '검증 층' 절](../USAGE.md)을 보세요.

## 실행 — 모든 스크립트 공통

여섯 스크립트는 실행 모양이 같습니다: **`python larp_○○_audit.py <입력파일> [옵션]` → 결정론 진단 출력 → 불일치가 있으면 종료코드 1.** 입력만 도구마다 다릅니다(원문 텍스트 · 1차 출력 마크다운 · 행렬/장부/통계 JSON). 셋 중 편한 길로 씁니다.

1. **가장 쉬움 — 그냥 시켜 보기.** 코드를 돌려 주는 AI(이 대화의 저, 또는 ChatGPT·Claude의 코드 실행)에 *입력 파일*을 주면 결과를 받습니다. **설치 필요 없음.**
2. **직접 돌리기.** 파이썬만 있으면 명령 한 줄. 각 스크립트는 `-h`로 인자를 확인합니다.
3. **코드 없이 — 챗봇용 근사.** 코드를 전혀 안 쓰려면 [`coverage_audit_prompt.md`](coverage_audit_prompt.md)를 챗봇에 붙여 씁니다. 단 *AI가 직접 읽는* 방식이라 **빠짐 없는 보장은 없습니다**(1·2번과 다른 점).

> **무판정 원칙.** 이 도구들은 옳고 그름을 가리지 않습니다 — 사람이 놓쳤을 불일치만 *보이게* 만들고, 판정은 사람의 몫입니다(LARP 원칙).
>
> **명령어·꼬리표 종류·운용 흐름을 상세히** 보려면 대표 예시 사용서 [`coverage_audit.md`](coverage_audit.md)를 보세요 — 나머지 스크립트도 같은 요령입니다.
