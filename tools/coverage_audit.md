*한국어 | [English](coverage_audit.en.md)*

> 긴 문서 모드의 *누락 방어*를 **프롬프트 지시**에서 **코드 대조**로 끌어올린 보조 도구입니다.
> 모델의 "우연히 찾음(LLM 읽기는 손실적)"을 **"인용 색인을 소진했나(결정론적)"** 로 바꿉니다.
> LARP은 판결문만 다루지 않으므로, 이 감사는 **인용 표지를 가진 어떤 문서에도** 통하도록 일반화돼 있습니다.

---

## 왜 필요한가

긴 글의 가장 깊은 누락은 *입력측 검색 누락* — 모델이 수백 페이지에서 어떤 참조를 **못 찾아** 애초에 안 내놓는 것이다. 이것은 *어떤 프롬프트로도(단일이든 에이전트든) 보장으로 막을 수 없다*. LLM의 읽기 자체가 손실적이기 때문이다. 보장에 가까워지는 유일한 길은, 모델의 기억·스캔에 의존하지 않고 **문서 자체의 인용 색인을 코드로 추출해 트리와 대조**하는 것이다.

## 작동 (3단계)

1. **인용 색인 추출(결정론).** 문서가 *인용 표지로 실제 인용한* 참조를 코드로 전수 추출한다. 쟁점 구간(줄 범위)으로 **스코프**할 수 있다.
2. **트리와 대조.** Map 트리가 각 참조를 다뤘는지(트리에 그 표지가 인용됐는지) 대조한다.
3. **누락 플래그.** 색인엔 있는데 트리엔 없는 참조를 `[누락?]`로 표시하고, 그 정체(문맥)를 함께 보인다 → coverage 장부에 반영.

## 지원하는 인용 표지 (도메인 일반)

기본은 **자동 감지**(가장 많이 인용된 표지 체계를 고름). 강제 지정은 `--scheme`, 임의 표지는 `--pattern`.

| 체계 | 대상 | 인식 형태 |
|---|---|---|
| `kr-judgment` | 한국 증거목록 | `증거목록 순번 N` · `순번 N` · `N 내지 M`(범위) |
| `bracket-num` | 번호 참고문헌(IEEE·밴쿠버) | `[12]` · `[12, 15]` · `[12-15]` |
| `exhibit` | 영미권 증거물 | `Exhibit 12` · `Exhibit A` · `Ex. 5` |
| `author-year` | 저자-연도 인용 | `(Smith 2020)` · `(Smith & Lee, 2019)` · `(Smith et al., 2021)` |
| `custom` | 사용자 정의 | `--pattern 'REGEX'` (캡처 그룹 1 = 키) — 각주 `fn. 3`, 조문 번호 등 무엇이든 |

## 보장과 한계 (정직히)

**보장:** *표지로 인용된* 참조는 트리에 있거나 `[누락?]`로 *표시된다*. 즉 인용 참조에 한해 **침묵의 누락 0**. (이게 "회의록·진술서 통째 누락" 같은 실패를 직접 잡는다.)

**한계:**
- 표지 없이 *이름으로만* 든 참조, 본문에 없는 **별지 마스터 목록**, 표지 없는 정황은 잡지 못한다.
- 정밀 대조를 위해 트리는 참조를 **같은 표지로 인용**해야 한다. 안 하면 보수적으로 *과다 플래그*(recall 우선이라 허용 가능 — 사람이 기각).
- 이것은 *판정*이 아니다. "다룸/누락?"은 *커버리지* 표시일 뿐, 참조의 진위·진단성과 무관하다(서기/판단자 경계 유지).

## 운용 — 긴 문서 모드와 결합

1. [긴 문서 모드](../prompts/LARP_map_long.md)의 **"출처·자료 견인"** 단계에서 트리가 참조를 *표지로* 인용하게 한다.
2. 매 쟁점(최종 결론)을 마칠 때 그 구간에 이 검사를 돌린다.
3. `[누락?]` 키를 트리에 `[미확장]`/`[누락?]` 노드로 되먹여 coverage 장부를 갱신한다.
4. 모든 키가 *다룸* 또는 *표시됨*이 될 때까지 반복 → 인용 참조 기준 무(無)침묵 누락.

## 사용법

```bash
# 인용 색인만 추출 (체계 자동 감지)
python larp_coverage_audit.py document.txt

# 체계 강제 / 쟁점 구간(줄 범위)만 / 임의 표지
python larp_coverage_audit.py document.txt --scheme bracket-num
python larp_coverage_audit.py document.txt --from 977 --to 1155
python larp_coverage_audit.py document.txt --pattern 'fn\.?\s*(\d+)'

# Map 트리와 대조(누락 감사) — 트리 텍스트를 파일로
python larp_coverage_audit.py document.txt --tree tree.txt
```

의존성 없음(파이썬 표준 라이브러리만). 색인은 `evidence_index.json`으로도 저장된다.

> **코드를 못 돌리는 환경?** 챗봇용 *근사* [`coverage_audit_prompt.md`](coverage_audit_prompt.md)가 있다 — AI가 직접 읽어 인용을 뽑으므로 **누락 보장은 없다**(코드판과 달리 "운 좋게 찾음"으로 되돌아감). 보장이 필요하면 위 코드판을 쓰되, 설치가 부담이면 *코드 실행되는 챗봇*에 스크립트를 올려 돌리면 설치 없이도 결정론을 유지한다.

## 실측 예 (수원고등법원 2024노620, "미화 총액" 쟁점)

이 쟁점 구간에서 순번으로 인용된 증거 = **239 · 326 · 726** 셋. 726(영수증)만 인용한 트리를 대조하면:

```
[audit] covered by tree: ['순번 726 ...']
[audit] ★ omission candidates (cited but absent from tree): ['순번 239 ...', '순번 326 ...']
  순번 239 | missing? | …진술하였다(2023고합815 증거목록 순번 239)…
  순번 326 | missing? | …LD의 진술서(2023고합185 증거목록 순번 326)…
```

→ 트리가 빠뜨린 LD 진술서(326)·전문 진술(239)을 **코드가 자동으로** 잡아낸다. 모델의 판단에 기대지 않는다. 같은 방식이 논문(`[15]` 누락)·계약 분쟁(`Exhibit B` 누락)에도 그대로 적용된다.
