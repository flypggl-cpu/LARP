# 전체판 사용법

*한국어 | [English](USAGE.en.md)*

> 이 문서는 **전체판**을 돌리고 그 결과를 읽는 법입니다. *짧은 글 빠른 점검*은 [라이트 사용법](USAGE_lite.md), *어떤 도구를 언제* 쓸지는 [README](README.md)에 있습니다.

**한 바퀴는 이렇습니다.** 프롬프트를 붙이고 → 분석할 글을 붙이면 → 도구가 논증을 펼쳐 *1차 분석*을 내고 **멈춥니다** → 내가 깊이 볼 곳을 고르면 → 그곳만 *2차*로 자세히 → **판단은 내가** 합니다. 설치는 없습니다.

---

## 1. 실행하기 — 5단계

1. **챗봇을 엽니다.** chatgpt.com이나 claude.ai에서 새 대화를 엽니다(무료로도 됩니다).
2. **프롬프트를 붙입니다.** [`prompts/LARP.md`](prompts/LARP.md) 내용을 *통째로* 복사해 첫 메시지로 붙입니다(길어도 괜찮습니다). 2차까지 갈 거면 [`prompts/LARP_modules.md`](prompts/LARP_modules.md)도 *이어서 함께* 붙입니다.
   - 자주 쓰면 챗봇의 '프로젝트'나 '맞춤 지시'에 한 번 넣어 두면 매번 붙일 필요가 없습니다.
3. **분석할 글을 붙입니다.** 다음 메시지로 *원문 전문*을 붙입니다(요약 말고 — 도구가 원문에서 논증을 뽑기 때문). "이 글의 결론이 타당한지 봐줘"처럼 한 줄 곁들이면 좋습니다. (더 정확히 하려면 아래 〈넣을 것〉 양식을 같이 적으세요.)
4. **1차 분석이 나오고 멈춥니다.** 도구가 *쉬운 요약 → 논리 지도 → 증거 대장*까지 내고 **멈춥니다**. 읽는 법은 **§2**. (안 멈추고 끝까지 가면 "1차까지만 하고 멈춰"를 한 줄 덧붙이세요.)
5. **깊이 볼 곳을 고르고 → 2차 → 내가 판단.** 더 파고들 곳(예: `A1`, `W1`)을 적으면 그곳만 자세히 봅니다(지정법 **§3**). 그냥 "계속"이라고 하면 핵심(★)만 알아서 봅니다. 도구는 "여기가 의심스럽다"고 *자리만* 짚고, 받아들일지는 사람이 정합니다.

> **긴 판결문은 통째로 넣지 마세요.** 죄목·쟁점이 여럿이면 *쟁점별로 나눠* 돌립니다 → **§4**.

**〈넣을 것〉 양식 (선택).** 글만 넣어도 되지만, 아래를 같이 적으면 더 정확합니다. 빈 칸은 도구가 "자료 없음"으로 표시하고 진행합니다.

```text
[분석 대상]   "○○ 제작자는 사기다"라는 온라인 고발 글
[검토할 주장] "제작자는 처음부터 만들 생각 없이 후원금을 가로챘다"
[가진 근거]   후원금 입금 내역, 미배송 사실, 돈을 어디에 썼는지 정황
[분석 목적]   주장 검토
```

## 2. 결과 읽는 법 — 1차에 무엇이 나오나

**맨 앞에 *쉬운 요약*(코드·전문용어 없는 평서문)이 먼저 나옵니다 — 그것부터 읽으세요.** "이 글의 결론은 ―, 몰래 기대는 가정은 ―, 결정적으로 보이는 근거가 정말 가르는가, 진짜 가르는 건 무엇이고 무엇이 비었는가"를 쉬운 말로 정리해 줍니다. 아래는 그 뒤에 붙는 *상세*를 읽는 법입니다.

### (1) 논증 블록
논증마다 짧은 재구성 블록. 먼저 볼 셋:

- **숨은 전제** — 글이 말 안 하고 몰래 기대는 가정(`암묵` 표시).
- **갈림** — 겉 이유와 실제 작동 이유가 다른 곳. 제일 먼저 볼 곳.
- **여섯 질문 결과** — "아니다 / 모르겠다"가 나온 번호. 의심의 이유가 거기 있습니다.

### (2) 논리 지도 (기호)

| 기호 | 뜻 |
|---|---|
| C / A / E | 결론 / 주장 / 증거 (글에 적힌 것) |
| W | 숨은 전제 (안 적힌 것, 점선) |
| L | 그렇게 보이게 만든 조건 (점선) |
| H | 다른 설명(대체가설) (점선) |
| V | 있어야 하는데 없는 증거 |
| ★ | 결론에 가장 중요한 것 (안 고르면 이것만 2차로) |

### (3) 증거 대장
인용·언급된 증거가 *하나도 안 빠지고* 한 줄씩. 칸 — *원천*(1차 직접 / 하류 전문 / 객관), *공통원천 묶음*(같은 출처면 한 번만 셈), *진단성*(아래 (4)), *원본성*(원본·증거능력 다툼). 도구가 "인용됐는데 안 다뤘다"고 짚은 **누락 후보**가 있으면 함께 표시됩니다 — *무엇이 비었는지*가 여기서 보입니다.

### (4) 증거×가설 행렬 (경쟁가설이 있을 때)
"이거냐 저거냐"가 갈리는 글이면 증거(행)×가설(열) 표가 나옵니다.

- 칸: `+` 부합 / `−` 배치 / `0` 무관 / `±` 읽기에 따라 갈림.
- **비진단** = 한 증거가 *여러 가설에 다 부합*(예: "돈을 지급했다"는 어느 설명에나 맞음) → 핵심 근거로 쓰면 약함.
- **독립 진단 지지** = *실제로 가르고*(한쪽에만 부합) *공통원천·전문이 아닌* 증거. 결론이 이걸 가졌는지가 관건.
- **빠진 증거(V)** = 그 가설이 맞다면 있어야 하는데 없는 것. 어느 가설에 구멍이 몰렸는지 보세요.

한마디로 — *그럴듯하게 맞음(정합성)이 아니라 가르는 힘(진단성)*으로 읽습니다.

### (5) 글 전체 의견 (3신호)
끝에 문서 전체 소견 — "세 신호(암묵 전제 쏠림·비진단 증거 의존·반증 안 되는 구조)가 다 한 방향이면, 증거를 모으기 전에 결론을 정해 둔 글일 수 있다"는 식.

→ 그다음 도구는 멈추고 "어디를 더 볼까요?"를 묻습니다(**§3**).

## 3. 깊이 볼 곳 지정 (2차)

지도를 보고 더 파고들 곳을 골라 적습니다.

```text
A1, W1, V1      ← 특정 항목 지정
E1->A1          ← 특정 연결 지정
논증 후보 2      ← 번호로 지정
계속            ← ★ 표시된 것만 알아서 진행
```

특정 문장 하나만 빠르게 보고 싶으면: `이 논증만 봐줘. 지도는 건너뛰고. "...원문 문장..."`

2차에서도 맨 앞에 *쉬운 요약*(무엇이 달라졌고 무엇이 남았나)이 먼저 나옵니다.

## 4. 큰 사건(긴 판결문)은 쟁점별로

전체판은 애초에 길고 복잡한 글을 위한 도구입니다 — 다만 판결문 한 건엔 여러 죄목과 수백 개의 증거가 있어, **통째로 붙이면 도구도 사람도 다 담지 못합니다**(AI는 긴 글 중간을 흘리고, 도구는 *핵심 주장 + 상위 몇 개*에 집중하도록 설계). 그래서 **쟁점(죄목)별로 나눠** 돌립니다.

1. **쟁점 하나를 고릅니다.** 죄목·쟁점 단위로 끊어(예: '뇌물 직무관련성', '대북송금 대납 여부') 그 판단 대목만 넣습니다.
2. **(아주 길면) 구조부터 펼칩니다.** [LARP-Map 긴 문서 모드](prompts/LARP_map_long.md)로 *결론에서 한 단계씩* 펼쳐 큰 그림을 잡습니다.
3. **그 쟁점을 §1대로 전체판에 돌립니다.** 증거 대장과, 경쟁가설이 있으면 증거×가설 행렬까지 나옵니다.
4. **빠진 곳은 결과에 이미 표시됩니다.** *빠진 증거(V)*·*누락 후보*를 도구가 스스로 짚어 줍니다 — 따로 안 돌려도 '무엇이 비었는지'는 보입니다.
5. **다음 쟁점으로, 마지막에 합칩니다.** 쟁점을 가로지르는 연결(한 사실이 두 죄목에 겹쳐 쓰임 등)은 사람이 잇습니다.

**"한 번에 모든 증거·모든 가설"은 도구의 약속이 아닙니다** — 한 번에 다 펼치는 건 사람도 못 합니다. 완전성은 *한 번의 실행*이 아니라 *쟁점별로 나눠 돌리는 절차*에서 나옵니다.

## 5. 더 정확히 — 선택 기능

평소엔 §1~§4로 충분합니다. 아래는 *더 확실히* 하고 싶을 때만.

### 5.1 바깥 확인이 필요할 때
글 하나로는 확인 못 하는 것(판례, 원본 대조 등)이 있으면, 도구는 단정하지 않고 **그대로 복사해 찾아볼 질문**을 만들어 줍니다 — 검색용은 *딥리서치 질문문*, 사건 기록용은 *"이 기록의 이 부분을 확인하라"는 지시*. 찾은 결과를 **출처와 함께** 다시 붙이면 반영합니다(출처 없는 결과는 사실로 안 받습니다).

### 5.2 주장 하나를 받아들일지 정할 때 (딥리서치 + LARP)
뉴스·SNS의 주장이 미심쩍을 때: ① 딥리서치로 근거를 모으고 → ② 주장+자료를 LARP에 넣어 숨은 전제·빠진 증거를 보고 → ③ 도구가 만들어 준 *추가 확인 질문*을 다시 딥리서치 → ④ 채택 여부는 내가 정합니다. (a) 딥리서치→LARP, (b) LARP→딥리서치 어느 쪽으로 돌아도 됩니다.
→ 예시: ["백신은 효과가 없다" 주장 검증](examples/claim_check_vaccine.md)

### 5.3 빠짐·환각까지 코드로 점검 — 검증 층 (선택)
분석 자체가 이미 *빠진 증거·누락 후보*를 내놓으니 평소엔 그걸로 충분합니다. 다만 **빠지면 큰일 나는 일**이면, 모델 혼자 못 잡는 두 위험(*안 올라온 누락*, *원문인 척 지어낸 인용*)을 바깥에서 한 번 더 봅니다.

1차 출력을 텍스트 파일로 저장한 뒤:

```bash
# ① 인용이 원문에 실제로 있나 (환각 탐지)
python tools/larp_quote_audit.py --source 원문.txt --analysis 1차출력.md
# ② 인용된 증거가 다 들어왔나 + 대장·카드 완전성 (누락 탐지)
python tools/larp_coverage_audit.py 원문.txt --tree 1차출력.md
python tools/larp_card_audit.py 1차출력.md
```

그리고 **누락 사냥 2차** — *새 창/다른 모델*에 [`prompts/LARP_verify.md`](prompts/LARP_verify.md)+원문+1차출력을 넣어 '안 올라온' 약점·증거·반론만 받습니다(같은 대화에서 시키면 자기 사각을 못 봅니다). 자세한 커버리지 옵션은 [tools/coverage_audit.md](tools/coverage_audit.md). **파이썬이 부담되면** 코드 실행되는 챗봇(ChatGPT·Claude)에 스크립트와 파일을 올려 돌리면 설치 없이 됩니다.

검증 층은 위험을 *없애지* 않고 *보이게* 만들 뿐 — 마지막 판단은 사람입니다.

## 6. 자주 묻는 것

**Q. 도구가 "이 글은 틀렸다"고 말해 주나요?** 아니요. "여기가 의심스러우니 살펴보라"고 *자리만* 짚습니다. 받아들일지는 사람이 정합니다.

**Q. 지적이 너무 많거나 적어요.** "좀 더 보수적으로 / 좀 더 공격적으로 골라줘"라고 하면 조절됩니다. 제외한 것의 이유도 함께 보여 주니 그 목록도 보세요.

**Q. 형사 사건 문서가 아니어도 되나요?** 됩니다. 기사·보고서·투자 판단, 내 생각에도 똑같이 씁니다. 형사 전용은 '합리적 의심 보고서' 하나뿐.

**Q. 결과가 너무 어려워요.** 맨 앞 *쉬운 요약*을 먼저 보세요. 그래도 어려우면 "중학생도 이해하게 쉽게 풀어서 설명해줘"를 덧붙이면 됩니다.

**Q. 1차에서 안 멈추고 끝까지 가버려요.** 첫 메시지에 **"1차(논증 지도)까지만 하고 멈춰"**를 덧붙이세요. 작은 모델일수록 이 한 줄이 도움이 됩니다.

**Q. 보안은?** **실제 사건기록·송치자료는 외부 AI에 넣지 마세요.** 연습은 공개된 글로.
