# 사용법 — 쉽게 따라 하기

*한국어 | [English](USAGE.en.md)*

> 어떤 도구를 *언제* 쓰는지(전체 흐름)부터 잡고 싶다면 → [사용 안내: 어떤 도구를 언제](docs/workflow.md). 이 문서는 그 도구들을 *실제로 어떻게 돌리는지*를 다룹니다.

## 1. 설치는 없다

LARP는 프로그램이 아니라 **글(프롬프트) 하나**입니다. [`prompts/LARP.md`](prompts/LARP.md)(영어판: [`LARP.en.md`](prompts/LARP.en.md))의 내용을 챗봇에 넣는 게 전부예요.

- **Claude · ChatGPT 같은 채팅 AI**: 프롬프트 전체를 첫 메시지로 붙이고, 이어서 분석할 글을 붙입니다. (프로젝트·맞춤 지시 기능이 있으면 거기 한 번 넣어두면 매번 붙일 필요가 없습니다.)
- **긴 글일수록** 한 번에 많이 읽는 큰 모델이 좋습니다.
- 1차 분석과 2차 분석은 **같은 대화 안에서** 이어가야 번호(A1, W1 등)가 맞물립니다.
- **전체판은 본문과 기준·점검 모듈 [`prompts/LARP_modules.md`](prompts/LARP_modules.md)을 함께 붙여** 1차·2차를 진행하세요. 1차(논증 지도)는 본문만으로도 되지만, 2차 상세 분석엔 모듈이 필요합니다.
- **모드 고르기 (권장 순서):** ① 먼저 **LARP-Map**([`prompts/LARP_map.md`](prompts/LARP_map.md))으로 전수 도해 → ② **전체판+모듈**로 깊이 분석. *길거나 복잡한(다주장) 글*은 이 흐름을 권합니다. ③ *짧고 단순한 글*만 **LARP-Lite**로 빠르게 점검(지름길). (Lite는 긴 글을 놓칩니다.)
  - *아주 길고 복잡한 글*(대형 판결문·다단계 논증)은 기본 Map이 한 번에 다 못 펼칩니다. 이때는 **LARP-Map 긴 문서 모드**([`prompts/LARP_map_long.md`](prompts/LARP_map_long.md))를 쓰세요 — 최종 결론부터 한 단계씩 대화형으로 펼치며, 매 턴 누적 트리와 coverage 장부로 *못 펼친 가지·누락 가능 영역*을 눈에 보이게 합니다(제로 누락 보장은 아님). 다 펼친 뒤 ②로 이어가세요.

## 2. 무엇을 넣나

최소한 **글 하나**면 됩니다. 더 정확하게 보고 싶으면 아래처럼 몇 가지를 같이 적어 주면 좋습니다.

> 입력은 요약이 아니라 **글 전문**을 넣으세요. 도구는 적힌 원문에서 논증 후보를 뽑기 때문에, 한 줄 요약만 넣으면 분석할 거리가 빈약해집니다. 한 줄짜리 주장 하나를 검증하고 싶다면, [§5.5 주장 검증 패턴](#55-활용-패턴-어떤-주장을-받아들일지-정할-때)처럼 딥리서치로 근거에 살을 붙인 뒤 넣는 게 좋습니다.

```text
[분석 대상]   "○○ 제작자는 사기다"라고 주장하는 온라인 고발 글
[보이는 것]   후원금을 받고 제품을 배송하지 못한 제작자
[붙은 이름]   처음부터 작정한 사기꾼
[검토할 주장] "제작자는 처음부터 만들 생각·능력 없이 후원금을 가로챘다"
[가진 근거]   후원금 입금 내역, 후원자 항의, 미배송 사실, 돈을 어디에 썼는지 정황
[분석 목적]   주장 검토
```

다 못 채워도 괜찮습니다. 비면 도구가 "이 부분은 자료 없음"이라고 표시하고 진행합니다.

## 3. 1차 결과 읽는 법

1차에서 도구는 깊은 분석을 하지 않습니다. **논증마다 짧은 요약 블록**과 **논리 지도**를 그려 주고 멈춥니다.

블록에서 먼저 볼 세 가지:

- **숨은 전제(warrant)** — 글이 말 안 하고 몰래 기대고 있는 가정. `암묵`이라고 표시됩니다.
- **갈림(대조)** — 겉으로 내세운 이유와 실제로 작동한 이유가 다를 때. 여기가 제일 먼저 봐야 할 곳입니다.
- **여섯 질문 결과** — "아니다 / 모르겠다"가 나온 질문 번호. 의심해야 할 이유가 거기서 나옵니다.

지도에 나오는 기호:

| 기호 | 뜻 | 비고 |
|---|---|---|
| C / A / E | 결론 / 주장 / 증거 | 글에 적힌 것 |
| W | 숨은 전제 | 안 적힌 것 (점선) |
| L | 그렇게 보이게 만든 조건 | 안 적힌 것 (점선) |
| H | 다른 설명(대체가설) | 안 적힌 것 (점선) |
| V | 있어야 하는데 없는 증거 | "꼭 필요/있으면 좋음" 등급 표시 |
| ★ | 결론에 가장 중요한 것 | 따로 안 고르면 이것만 2차로 넘어감 |

지도 다음에는 **글 전체에 대한 의견**이 붙습니다. "세 가지 신호가 다 한 방향이면, 이 글은 증거를 모으기 전에 결론을 정해 둔 글일 수 있다"는 식입니다.

## 4. 깊이 볼 곳 고르기 (2차)

지도를 보고 더 파고들 곳을 골라 적으면 됩니다.

```text
A1, W1, V1      ← 특정 항목 지정
E1->A1          ← 특정 연결 지정
논증 후보 2      ← 번호로 지정
계속            ← ★ 표시된 것만 알아서 진행
```

특정 문장 하나만 빠르게 보고 싶으면:

```text
이 논증만 봐줘. 지도는 건너뛰고. "...원문 문장..."
```

## 5. 바깥에서 확인이 필요할 때

글 하나만으로는 확인할 수 없는 것(판례, 원본 자료 대조 등)이 있습니다. 이때 도구는 함부로 단정하지 않고, **그대로 복사해서 찾아볼 수 있는 질문**을 만들어 줍니다.

- **검색으로 찾을 수 있는 것** → 딥리서치 AI에 넣을 질문문을 만들어 줍니다.
- **사건 기록에서만 나오는 것** → "이 기록의 이 부분을 확인하라"는 지시를 만들어 줍니다.

찾은 결과를 **출처와 함께** 대화에 다시 붙이면, 도구가 그걸 반영합니다. 출처 없는 결과는 사실로 받아들이지 않습니다(인공지능이 지어낸 말을 거르기 위해서예요).

## 5.5 활용 패턴: 어떤 주장을 받아들일지 정할 때

뉴스·SNS·보고서에서 본 주장 하나가 미심쩍을 때, 딥리서치와 함께 이렇게 쓰면 좋습니다.

1. **딥리서치로 근거를 모은다.** 그 주장이 기대는 사실·통계·출처를 딥리서치 AI(또는 검색)로 모읍니다.
2. **주장과 모은 자료를 LARP에 넣는다.** 도구가 그 주장의 논증을 펼쳐, 숨은 전제·다른 설명·빠진 증거를 보여 주고, 근거가 결론에 실제로 닿는지 따집니다.
3. **무엇을 더 확인할지 도구가 알려준다.** LARP는 단정 대신 "이걸 확인해야 한다"는 딥리서치 질문을 직접 만들어 줍니다. 그 답을 다시 넣으면 분석이 갱신됩니다.
4. **채택 여부는 내가 정한다.** 도구는 "받아들여라/버려라"를 말하지 않습니다. 대신 그 주장의 근거가 어디까지 버티는지, 무엇이 아직 비었는지를 보여 주고, 결정과 책임은 사람에게 남깁니다.

순서는 두 방향으로 돌 수 있습니다. **(a)** 먼저 딥리서치 → 그 결과를 LARP로 분석, 또는 **(b)** 먼저 LARP → 도구가 만들어 준 질문을 딥리서치 → 결과를 다시 넣기. 둘을 오가면, "그럴듯해 보이는 주장"을 "근거가 버티는 주장"과 구별할 수 있습니다. 이건 가짜뉴스·가짜 분석이 넘치는 환경에서 특히 쓸모 있습니다 — 결론을 대신 내려 주는 게 아니라, 받아들일지 말지를 **내가 근거 위에서** 정하게 도와주니까요.

→ 구체적 예시: ["백신은 효과가 없다" 주장 검증](examples/claim_check_vaccine.md)

## 5.6 긴 글의 누락 증거 검사 — 결정론적 (coverage audit, 선택)

> 한글로는 **누락 증거 검사**, 영어 명칭은 **coverage audit**입니다.

긴 문서 모드로 트리를 그려도, 모델이 수백 페이지에서 어떤 증거를 *못 찾아* 애초에 안 내놓는 누락은 프롬프트로는 막지 못합니다(LLM 읽기는 손실적). 이때 보조 스크립트 [`tools/larp_coverage_audit.py`](tools/larp_coverage_audit.py)가 *문서가 표지로 인용한* 참조를 코드로 전수 추출해 트리와 대조하고, 트리에 빠진 인용을 `[누락?]`로 짚어 줍니다 — **인용 참조에 한해 침묵의 누락 0**. (이 한 부분만 파이썬 실행이 필요하고, 나머지 LARP은 설치가 필요 없습니다.)

쓰는 법:

```bash
# 1) 문서에서 인용 색인만 추출 (표지 체계 자동 감지)
python tools/larp_coverage_audit.py document.txt

# 2) 쟁점 구간(줄 범위)만 보기
python tools/larp_coverage_audit.py document.txt --from 977 --to 1155

# 3) Map 트리와 대조 — 긴 문서 모드의 트리를 텍스트 파일로 저장해 넣기
python tools/larp_coverage_audit.py document.txt --tree tree.txt
```

표지 체계는 한국 증거목록(`순번 N`)·번호 참고문헌(`[12]`)·영미권 `Exhibit`·저자-연도 `(Smith 2020)`를 **자동 감지**하고, 그 밖의 표지는 `--pattern '정규식'`으로 지정합니다(예: 각주 `--pattern 'fn\.?\s*(\d+)'`). 운용 흐름과 한계는 [tools/coverage_audit.md](tools/coverage_audit.md)에 자세합니다.

**파이썬이 부담되면:** ① 코드 실행되는 챗봇(ChatGPT Advanced Data Analysis, Claude 등)에 스크립트와 문서를 올려 돌리면 설치 없이도 결정론이 유지됩니다. ② 코드를 아예 안 쓰려면 챗봇용 *근사* 프롬프트 [`tools/coverage_audit_prompt.md`](tools/coverage_audit_prompt.md)를 붙여 쓸 수 있습니다 — 단 AI가 직접 읽어 뽑으므로 **누락 보장은 없습니다**(코드판과 다른 점).

**한계:** 표지 없이 *이름으로만* 든 참조나 본문에 없는 별지 목록은 못 잡습니다. 그리고 이것은 진위·진단성 *판정이 아니라* "다뤘나/빠졌나"의 **커버리지 표시**일 뿐입니다.

## 6. 자주 묻는 것

**Q. 도구가 "이 글은 틀렸다"고 말해 주나요?**
아니요. 도구는 "여기가 의심스러우니 살펴보라"고 자리만 짚어 줍니다. 결론을 받아들일지는 사람이 정합니다.

**Q. 지적이 너무 많거나 적어요.**
"좀 더 보수적으로(엄격하게) / 좀 더 공격적으로 골라줘"라고 말하면 조절됩니다. 도구는 제외한 것의 이유도 함께 보여 주니 그 목록도 같이 보세요.

**Q. 형사 사건 문서가 아니어도 되나요?**
됩니다. 뉴스 기사, 보고서, 투자 판단, 심지어 내 자신의 생각에도 똑같이 쓸 수 있습니다. 형사 전용 기능은 '합리적 의심 보고서' 하나뿐입니다.

**Q. 결과가 너무 어려워요.**
"중학생도 이해할 수 있게 쉽게 풀어서 설명해줘"라고 덧붙이면 됩니다.

**Q. 1차에서 안 멈추고 끝까지 분석해 버려요.**
모델에 따라 1차와 2차를 한 번에 쏟아낼 때가 있습니다. 첫 메시지에 **"1차(논증 지도)까지만 하고 멈춰"**를 덧붙이면 확실합니다. 프롬프트에도 멈추라는 규칙이 들어 있지만, 작은 모델일수록 이 한 줄이 도움이 됩니다.
