# Contributing

*[한국어](CONTRIBUTING.md) | English*

Thank you for your interest in this project.

First, to be clear: **this repository is a methodology tool, not software.** It's one prompt (`prompts/`) plus documentation (`docs/`, `examples/`), with no code to build or run. So contributing requires no programming skill — what it needs is **an eye for finding flaws in reasoning.**

I (the maintainer) am not a developer but a practicing lawyer. So there's no elaborate process here. Please feel free to share your thoughts.

## Contributions welcome

- **Pointing out gaps in the arguments** — missing premises, overlooked alternative explanations, or weak links in the prompt's analysis steps or examples. This is the heart of the tool.
- **Fixing errors and typos** — factual mistakes, broken links, awkward wording.
- **Translation** — into languages beyond English, or polishing the existing Korean/English.
- **Use cases** — examples from applying it to new areas (articles, reports, investment decisions, etc.).
- **Plainer explanations** — anywhere that could be made easier to understand.

## How

It doesn't have to be formal.

1. **Open an Issue** — the easiest path. "This premise seems missing here," "this link is broken," "this part is hard to follow" is plenty.
2. **Pull request** — if you'd like to propose an edit directly, send a PR. Editing a file on the GitHub website creates a PR automatically, so you don't need to know any commands.

Discussion in Korean or English is equally welcome. Each document comes as a pair of language files (`*.md` / `*.en.md`); if you change one, please glance at the other too.

## One request

The core of this tool is **putting the same questions to everyone equally, regardless of side.** Rather than pushing a particular conclusion, please aim contributions at making the *structure* of the analysis more accurate and fair.
