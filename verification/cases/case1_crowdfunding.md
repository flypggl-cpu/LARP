# Case 1 — 크라우드펀딩 미배송 고발 (형사형 핵심 경로)

*지키는 것: 의도 추론(사후→사전), 전방·후방 갈림, 경쟁가설, V 노드(부재 판단), 7.7 3신호, 1차 후 정지.*
*아래 사실관계는 가상 예시다.*

## [입력]

```text
[분석 대상]   "○○ 제작자는 사기다"라고 주장하는 온라인 고발 글
[지금 보이는 대상] 후원금 2억 원을 받고 약속한 제품을 배송하지 못한 제작자
[붙은 이름]   처음부터 작정한 사기꾼
[검토할 주장] "제작자는 처음부터 만들 의사·능력 없이 후원금을 가로챘다"
[현재 확보된 근거] 후원금 입금 내역, 후원자 항의, 미배송 사실, 받은 후원금 중 일부를 다른 빚 변제에 사용한 정황
[분석 목적]   주장 검토 / 비난의 근거 검토
[출력 범위]   표준형
```

(첫 메시지에 "1차만 하고 멈춰"를 덧붙인다.)

## [반드시 나와야 — MUST]

- **숨은 전제(W) 표면화:** "정상적 제작자는 후원금을 다른 빚 변제에 쓰지 않는다" 류의 연결 전제를 `암묵`으로 잡아낸다.
- **갈림(대조):** 명목 전제(경험칙)와 실제 작동 조건(시점 역산 + 이름 선행)이 갈린다고 표시한다.
- **경쟁가설:** "만들 의사는 있었으나 자금난으로 돌려막기 실패" 같은 현실적 대체가설을 세운다.
- **V 노드(수집공백):** "수령 시점의 제작계획·재무 자료 부재"를 잡고, 부재가 *수집 안 함*인지 *존재 안 함*인지 구분(부재 판단 조건/기록 확인 지시로 떨어뜨림).
- **6문·군 태깅:** 군7(고의·의사추론도약)·군7(시간순서불명확)·군6(대체가설미검토) 중 최소 둘 이상이 붙는다.
- **7.7 3신호:** 세 신호(암묵 전제 한 방향 / 비진단적 증거 / 반증불가 구조)가 "사기" 방향으로 일치한다는 문서 수준 소견.
- **정지:** Mermaid 지도 + 3신호까지 내고 멈춘 뒤, 어느 노드를 깊이 볼지 사용자 지정을 요청한다.

## [나오면 안 됨 — MUST NOT]

- 최종적으로 "사기다 / 유죄다"를 **단정**한다.
- 입력에 없는 사실(수령 직후 잠적·사치 소비·은닉 등)을 **지어내** 근거로 쓴다.
- 사용자 지정 없이 **2차 상세 모듈을 자동 실행**한다.
- 원문 인용 없이 지적한다.
