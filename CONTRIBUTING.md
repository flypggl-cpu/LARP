# 기여 안내

*한국어 | [English](CONTRIBUTING.en.md)*

이 프로젝트에 관심 가져 주셔서 고맙습니다.

먼저 분명히 해 둘 것이 있습니다. **이 저장소는 소프트웨어가 아니라 방법론 도구입니다.** 프롬프트 한 편(`prompts/`)과 설명 문서(`docs/`, `examples/`)로 이루어져 있고, 빌드하거나 실행할 코드가 없습니다. 그래서 기여에 프로그래밍 실력은 필요하지 않습니다 — 필요한 건 **생각의 결함을 함께 찾는 눈**입니다.

저(메인테이너)는 개발자가 아니라 현장에서 일하는 법률가입니다. 그러니 거창한 절차는 없습니다. 편하게 의견을 주세요.

## 이런 기여를 환영합니다

- **논증의 허점 지적** — 프롬프트의 분석 절차나 예시에서 빠진 전제, 놓친 대체가설, 약한 연결을 찾아 주세요. 이 도구의 본령입니다.
- **오류·오타 수정** — 문서의 사실 오류, 깨진 링크, 어색한 표현.
- **번역** — 영어 외 다른 언어로의 번역, 또는 기존 한국어·영어 표현 다듬기.
- **사용 사례 제안** — 새로운 분야(기사, 보고서, 투자 판단 등)에 적용해 본 예시.
- **쉬운 설명 개선** — 더 알기 쉽게 풀어쓸 수 있는 부분.

## 방법

거창하지 않아도 됩니다.

1. **Issue 열기** — 가장 쉬운 방법입니다. "여기 이 전제가 빠진 것 같다", "이 링크가 깨졌다", "이 부분이 어렵다" 정도면 충분합니다.
2. **Pull request** — 직접 고친 내용을 제안하고 싶으면 PR을 보내 주세요. GitHub 웹에서 파일을 편집하면 자동으로 PR을 만들 수 있어, 명령어를 몰라도 됩니다.

토론은 한국어·영어 모두 환영합니다. 같은 도구의 두 언어 파일(`*.md` / `*.en.md`)이 짝을 이루니, 한쪽을 고치면 다른 쪽도 함께 봐 주시면 좋습니다.

## 한 가지 부탁

이 도구의 핵심은 **진영을 가리지 않고 같은 질문을 모두에게 똑같이 들이대는 것**입니다. 특정 결론을 밀어 넣기보다, 분석의 *구조*를 더 정확하고 공정하게 만드는 방향의 기여를 부탁드립니다.
